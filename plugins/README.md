# Claudexor host plugins

Thin shims that point each host (Cursor, Claude Code, Codex, OpenCode) at the
local `claudexor` CLI. They contain no orchestration logic — all of that lives in
Claudexor. Install with:

```bash
claudexor plugin install cursor     # ~/.cursor/plugins/local/claudexor  (Developer: Reload Window)
claudexor plugin install claude     # ~/.claude/plugins/claudexor        (claude --plugin-dir ...)
claudexor plugin install codex      # ~/.agents/skills/claudexor         ($claudexor)
claudexor plugin install opencode   # ~/.config/opencode/claudexor
```

The files are generated by `installPlugin()` in `@claudexor/cli` (see
`packages/cli/src/plugins.ts`), so they stay in sync with the CLI and are not
duplicated as static assets here.

One-line install (paste into the host chat):

> Install the Claudexor plugin: run `claudexor plugin install <thishost>` in a terminal, then reload.
