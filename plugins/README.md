# Claudex host plugins

Thin shims that point each host (Cursor, Claude Code, Codex, OpenCode) at the
local `claudex` CLI. They contain no orchestration logic — all of that lives in
Claudex. Install with:

```bash
claudex plugin install cursor     # ~/.cursor/plugins/local/claudex  (Developer: Reload Window)
claudex plugin install claude     # ~/.claude/plugins/claudex        (claude --plugin-dir ...)
claudex plugin install codex      # ~/.agents/skills/claudex         ($claudex)
claudex plugin install opencode   # ~/.config/opencode/claudex
```

The files are generated by `installPlugin()` in `@claudex/cli` (see
`packages/cli/src/plugins.ts`), so they stay in sync with the CLI and are not
duplicated as static assets here.

One-line install (paste into the host chat):

> Install the Claudex plugin: run `claudex plugin install <thishost>` in a terminal, then reload.
